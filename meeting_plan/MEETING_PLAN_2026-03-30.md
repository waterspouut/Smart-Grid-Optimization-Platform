# SGOP 2026-03-30 개발 회의안

## 1. 이번 회의 목표
- 오늘 회의는 아이디어 회의가 아니라 `범위 확정 + 역할 배정 + 1주차 산출물 확정` 회의다.
- 일정 기준은 `2026년 5월 10일 MVP`, `2026년 5월 24일 완성본`이다.
- `Optimization/ESS`는 MVP 필수 범위에서 제외하고 `2026년 5월 11일` 이후 확장 후보로 둔다.

## 2. MVP 기준
- `Monitoring` 동근: 혼잡도 수치와 차트, 위험 상태가 나온다.
- `Simulation` 나현: 후보지 입력 후 설치 전후 비교 결과가 나온다.
- `Prediction` 도림: 예측 그래프와 위험도 결과가 나온다.
- `지도 시각화`: 후보지, 경로, 상태 변화가 보인다.
- 외부 API나 데이터가 실패해도 `mock data`로 데모가 끊기지 않는다.

## 3. 최종 역할 배정표
| 담당자 | 메인 역할 | 세부 책임 |
|---|---|---|
| 박차오름 |공통 코어/통합 메인 | `A*`, 점수화 로직, 공통 인터페이스 설계, 서비스 간 연결, 최종 통합, fallback 결정, 병목 해결 |
| Alpha |Monitoring 메인 | `dc_power_flow`, `congestion_metrics`, 모니터링 페이지, 모니터링 서비스, KPI 정의, 차트/표/상태 시각화 |
| Beta |Simulation 메인 | 후보지 입력 UI, 설치 전후 비교 UI, 시나리오 흐름, 지도 오버레이, 시뮬레이션 서비스 연결 |
| Gamma |Prediction 메인 | `feature_builder`, baseline 또는 `LSTM`, 예측 페이지, 예측 서비스, 위험도/설명 출력, 테스트/QA 리드 |

## 4. 역할 분배 의도
- 박차오름은 모든 걸 혼자 만드는 사람이 아니라 `공통 코어와 통합 병목`을 잡는 중심 역할이다.
- Alpha는 단순 페이지 담당이 아니라 `혼잡도 계산`을 들고 가는 메인 오너다.
- Beta는 발표에서 가장 눈에 띄는 `시뮬레이션 경험 전체`의 메인 오너다.
- Gamma는 `AI가 들어간다`는 걸 가장 직접적으로 보여주는 예측 파트의 메인 오너다.

## 6. 2026-03-30 회의 진행 순서
1. 목표 확인, 5분
   - 오늘은 범위와 역할을 확정하는 회의라고 먼저 못 박는다.
2. MVP 정의, 10분
   - Monitoring, Simulation, Prediction, 지도 시각화까지를 MVP로 확정한다.
3. 제외/후순위 항목 정리, 10분
   - ESS 고도화, 정교한 3D, 고급 설명 기능은 후순위로 둔다.
4. 1주차 작업 확정, 20분
   - 각자 금요일에 보여줄 산출물 1개를 말하게 한다.
5. fallback 규칙 확정, 10분
   - 3D 실패 시 2.5D, LSTM 지연 시 baseline으로 간다.
6. 운영 규칙 확정, 10분
   - 일일 체크인, 주간 데모, 코드 리뷰, 막힘 공유 규칙을 정한다.

- “이번에는 공평 분배보다 속도 최적화 분배로 가겠습니다.”
- “저는 공통 코어와 통합을 맡고, 각자 메인 템 하나씩은 확실히 들고 가는 구조로 가겠습니다.”
- “모니터링은 혼잡도 계산, 시뮬레이션은 결과 비교, 예측은 AI 파이프라인이 각자의 메인입니다.”
- “저는 A*와 통합을 맡아서 병목을 줄이겠습니다.”
- “우리는 코드 양이 아니라 금요일에 보여줄 수 있는 데모로 진도를 판단하겠습니다.”
- “정확도보다 먼저 중요한 건 끊기지 않는 파이프라인입니다.”

## 8. fallback 규칙
- 3D가 `2026-04-05`까지 안정적으로 안 되면 `2.5D`로 전환한다.
- LSTM이 지연되면 baseline 예측으로 먼저 연결한다.
- 외부 API가 불안정하면 mock data로 데모를 유지한다.

## 9. 이 배정안의 장점
- 누가 놀게 되는 구조가 아니라, 각자 데모에 직접 보이는 결과물을 만든다.
- 최종 통합 시 책임 구간이 분명해서 병목이 줄어든다.

## 10. 조정 원칙
- 기본 배정은 `Alpha=Monitoring`, `Beta=Simulation`, `Gamma=Prediction`이다.
- 만약 팀원 선호가 다르면 `Alpha/Beta/Gamma`끼리 슬롯만 바꾼다.
